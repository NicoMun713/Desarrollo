Test Automation

Testing Automation It is a current approach that is on everyone's lips, since it allows tests to be carried out automatically and repetitively. Automation is implemented in fixed and stable sections, which are not going to be modified in the short term and repeatable precision at the speed of a machine is required.

Thanks to this possibility, companies can detect errors in their projects in time -and gain time to correct them-, as well as save significant resources. Likewise, by eliminating routine tasks, this method frees up time to carry out deeper actions related to software development and implementation. Automated testing is the recommended method when the solution is stable, customer experience is a high priority and time-to-market is critical or when repeatable accuracy is privileged, that is, that ability that machines have, and not so much people, to repeat a process as many times as necessary without losing attention.

Advantage

It is often believed that automated testing will end up making manual testing disappear. But nothing further than this statement. “Automation is not a replacement for manual testing, but rather a complementary process”

As its name indicates, manual tests are carried out by a person who, based on test cases, handles himself as a user would in order to detect failures and report them. Manual tests are mainly used for functional, usability and/or exploratory tests.

Generalidades

  • Pruebas manejadas por el código: Se prueban las interfaces públicas de las clases, módulos o bibliotecas con una variedad amplia de argumentos de entrada y se valida que los resultados obtenidos sean los esperados.
  • de Interfaz de Usuario: Un marco de pruebas genera un conjunto de eventos de la interfaz de usuario, tales como teclear, hacer click con el ratón e interactuar de otras formas con el software y se observan los cambios resultantes en la interfaz de usuario, validando que el comportamiento observable del programa sea el correcto.
